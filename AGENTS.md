# AGENTS.md — AI Coding Agent Instructions

This file contains instructions for AI coding agents (Cursor, Copilot, Claude, etc.)
working on this repository.

## Repository Overview

**dev-patterns** — central patterns library for orchestras repositories.

Provides: git hook management via declarative TOML manifests · patterns channel
subscriptions · mise task distribution · version-spec resolution with legacy support.

Stack: **UV** (packaging) · **Ruff** (lint + format) · **Ty** (type checking) · **pytest** (testing) · **Mise** (tasks + tools)

## Important Rules

1. **Never edit `src/dev_patterns/version.py` manually** — it is generated by `mise run build`
2. **Never merge branches** — always rebase: `mise run vcs:rebase`
3. **Version bumps** use `mise run bump:patch/minor/major` which commits + creates git tags
4. **Push tags separately** with `mise run tag:push` after all commits are ready

## Running Tasks

```bash
mise run install        # install all dependencies
mise run ci:all         # full CI pipeline
mise run lint           # ruff check
mise run lint:fix       # ruff check --fix
mise run fmt            # ruff format
mise run fmt:check      # ruff format --check
mise run typecheck      # ty check
mise run test           # pytest
mise run test:cov       # pytest + coverage
mise run bump:patch     # bump patch version
mise run bump:minor     # bump minor version
mise run bump:major     # bump major version
mise run tag:push       # push tags → triggers release
mise run patterns:sync  # sync patterns from dev-patterns channel
mise run hooks:sync     # sync git hooks
mise run scan:deps      # audit dependencies
mise run scan:secrets   # scan for leaked secrets
```

## Adding New Tasks

Create a new file in `.mise/tasks/<name>` with a bash shebang and the
`# mise description="..."` comment on line 2. Make it executable:
```bash
chmod +x .mise/tasks/<name>
```

For namespaced tasks (e.g. `foo:bar`), create `.mise/tasks/foo:bar`.

## Code Quality Standards

- All public functions must have **Google-style docstrings**
- All functions must have **type annotations**
- Minimum Python version: **3.12**
- Line length: **100**
- Run `mise run ci:all` before committing

## Security

- Use `mise run scan:deps` to audit dependencies before releases
- Use `mise run scan:secrets` to check for leaked secrets
- GHAS (CodeQL) runs weekly and on every push to main/develop
- Bandit runs on every PR as part of the security check
- Dependency Review requires Dependency Graph enabled in repo settings

## Git Workflow (rebase-only — no merge commits)

### Keeping a feature branch current

```bash
# From your feature branch:
mise run vcs:rebase
# Runs: git pull --rebase origin develop
```

### Integrating a feature branch into develop

```bash
mise run vcs:integrate feat/my-feature
# 1. git checkout feat/my-feature
# 2. git pull --rebase origin develop      (ensure feature is current)
# 3. git checkout develop && git pull      (ensure develop is current)
# 4. git rebase feat/my-feature            (fast-forward develop onto feature)
# 5. git push --force-with-lease origin develop
```

### Releasing develop → main

```bash
mise run vcs:release
# 1. git checkout main
# 2. git rebase origin/develop
# 3. git push --force-with-lease origin main
```

### Tagging and triggering a release

```bash
mise run bump:patch   # (or :minor / :major) — updates pyproject.toml, version.py, commits, tags
mise run tag:push     # pushes tag → triggers binary + Docker release workflow
```
